# 대시보드 차트 시각화 PRD

## 1. 배경

### 현재 상태

대시보드는 6개 섹션의 **표(테이블) 형식**으로만 데이터를 제공하고 있다. Google Sheets API의 차트 기능을 전혀 사용하지 않고 있으며, `google_sheets_client.py`에 차트 관련 메서드가 없다.

| 섹션 | 현재 제공 방식 | 시각화 한계 |
|------|---------------|------------|
| 섹션 1: 포트폴리오 요약 | 숫자 7개 (1행) | 한눈에 상태를 판단하기 어려움 |
| 섹션 2: 월별 성과 | (연월, 계좌) 테이블 | 추이 파악이 불가, 행이 많으면 비교 곤란 |
| 섹션 3: 투자 지표 | 지표명-값 2컬럼 | 비중(%)을 숫자로만 봐서 직관적이지 않음 |
| 섹션 4: 매매 인사이트 | 지표명-값 2컬럼 | 요일별 성과 등 패턴 파악 어려움 |
| 섹션 5: 월별 성과 추이 | 11컬럼 테이블 | **핵심 문제**: 추이 데이터인데 차트 없음 |
| 섹션 6: 종목별 현황 | 11컬럼 테이블 | 종목 간 비교가 숫자 나열로만 가능 |

### 핵심 문제

**숫자만으로는 추세와 패턴을 빠르게 판단할 수 없다.**

- 월별 성과 추이(섹션 5)에 10개월 데이터가 있을 때, 실현손익이 개선되고 있는지 테이블만으로 즉시 판단하기 어렵다
- 투자 비중(계좌별, 통화별)이 숫자(%)로만 제공되어 편중 여부를 직관적으로 파악할 수 없다
- 요일별 성과가 텍스트로 나열되어 패턴을 놓치기 쉽다

### 기대 효과

차트를 추가하면:
- **즉각적 추세 파악**: 월별 손익/승률 그래프로 성장 또는 악화를 즉시 확인
- **비중 시각화**: 파이 차트로 포트폴리오 편중 여부를 한눈에 파악
- **패턴 발견**: 요일별 성과 바 차트로 매매 타이밍 최적화
- **의사결정 지원**: 시각적 정보가 추가되어 전략 조정 판단이 빨라짐

## 2. 요구사항

### 2.1 차트 배치 전략

대시보드 시트 우측(N열 이후)에 차트 영역을 배치한다. 기존 데이터 테이블(A~K열)과 겹치지 않도록 한다.

```
A ──────────── K    N ──────────── X
[기존 테이블 영역]    [차트 영역]
섹션 1~6 데이터       차트 1~6 배치
```

### 2.2 차트 목록

#### 차트 1: 월별 실현손익 추이 (Column Chart)

- **데이터 소스**: 섹션 5 (월별 성과 추이) - A열(연월), C열(실현손익)
- **차트 유형**: 세로 막대 차트 (Column Chart)
- **목적**: 월별 수익/손실 규모를 시각적으로 비교
- **포맷**:
  - X축: 연월 (YYYY-MM)
  - Y축: 실현손익(원), `₩#,##0` 포맷
  - 양수: 파란색, 음수: 빨간색 (조건부 색상 적용이 API에서 지원되면)
- **크기**: 600 x 370 px

#### 차트 2: 월별 승률 & 수익률 추이 (Line Chart)

- **데이터 소스**: 섹션 5 - A열(연월), D열(수익률), E열(승률)
- **차트 유형**: 라인 차트 (2개 시리즈)
- **목적**: 수익률과 승률의 시간에 따른 변화를 추적
- **포맷**:
  - X축: 연월
  - Y축: 백분율(%), `0.0%` 포맷
  - 수익률: 파란색 라인, 승률: 녹색 라인
  - 범례 표시
- **크기**: 600 x 370 px

#### 차트 3: 계좌별 투자비중 (Pie Chart)

- **데이터 소스**: 섹션 3 (투자 지표) - "계좌별 투자비중" 소섹션의 행 데이터
- **차트 유형**: 파이 차트
- **목적**: 계좌 간 자산 배분 현황을 한눈에 파악
- **포맷**:
  - 각 조각: 계좌명 + 비율(%) 레이블
  - 색상: Google Sheets 기본 팔레트
- **크기**: 450 x 370 px

#### 차트 4: 요일별 실현손익 (Column Chart)

- **데이터 소스**: 섹션 4 (매매 인사이트) - "요일별 성과" 소섹션
- **차트 유형**: 세로 막대 차트
- **목적**: 어떤 요일에 성과가 좋은지/나쁜지 패턴 파악
- **포맷**:
  - X축: 요일 (월~금)
  - Y축: 실현손익(원)
- **크기**: 450 x 370 px

#### 차트 5: 손익비 & Profit Factor 추이 (Line Chart)

- **데이터 소스**: 섹션 5 - A열(연월), H열(손익비), I열(Profit Factor)
- **차트 유형**: 라인 차트 (2개 시리즈)
- **목적**: 매매 효율성 지표의 시계열 추이 확인
- **포맷**:
  - X축: 연월
  - Y축: 비율 (0.00)
  - 손익비: 주황색, Profit Factor: 보라색
  - 기준선: 1.0 (손익비·PF 모두 1 이상이 양호)
- **크기**: 600 x 370 px

#### 차트 6: 수익 Top 10 종목 (Bar Chart)

- **데이터 소스**: 섹션 3 (투자 지표) - "수익 Top 10" 소섹션
- **차트 유형**: 가로 막대 차트 (Bar Chart)
- **목적**: 어떤 종목이 가장 큰 수익을 냈는지 한눈에 비교
- **포맷**:
  - Y축: 종목명
  - X축: 실현손익(원)
  - 색상: 파란색 계열
- **크기**: 600 x 370 px

### 2.3 차트 배치 위치

차트를 세로로 나열하여 스크롤하며 확인할 수 있도록 배치한다.

```
행 위치 (대략)       N열~           차트
1~20행              차트 1         월별 실현손익 추이
21~40행             차트 2         월별 승률 & 수익률 추이
41~60행             차트 3 + 4     계좌별 투자비중 | 요일별 실현손익
61~80행             차트 5         손익비 & Profit Factor 추이
81~100행            차트 6         수익 Top 10 종목
```

> **참고**: 정확한 행 위치는 앵커 셀 (offsetXPixels, offsetYPixels)로 조정. 테이블 데이터 영역과 겹치지 않도록 N열(13번째 컬럼) 이후에 배치.

### 2.4 차트 초기화 정책

대시보드 재생성 시 **기존 차트를 모두 삭제 후 재생성**한다.

- `_ensure_dashboard_sheet()`에서 기존 차트 목록 조회 → 전체 삭제
- 데이터 작성 완료 후 차트 생성

### 3.1 google_sheets_client.py - 신규 메서드

```python
{
    "addChart": {

                    "legendPosition": "BOTTOM_LEGEND",
                    "axis": [...],
                    "domains": [{

                                }]
                            }
                        }
                    }],
                    "series": [{
                        "series": {
                            "sourceRange": {
                                "sources": [{
                                    "sheetId": sheet_id,
                                    "startRowIndex": start,
                                    "endRowIndex": end,
                                    "startColumnIndex": 2,  # C열
                                    "endColumnIndex": 3,
                                }]
                            }
                        },

                }
            },
            "position": {
                "overlayPosition": {
                    "anchorCell": {

                    "heightPixels": 370,

```

### 3.2 summary_generator.py - 변경 사항

#### 데이터 위치 추적

차트가 참조하는 데이터 범위(행 인덱스)를 정확히 알아야 하므로, `generate_all()`에서 각 섹션의 시작/종료 행을 차트 생성 메서드에 전달한다.

```python
async def generate_all(self, all_trades: List[Trade]):
    # ... 기존 섹션 작성 ...

    # 차트 생성 (기존 차트 삭제 후)
    await self._delete_existing_charts()
    await self._create_charts(
        trend_start=trend_start,
        trend_end=current_row,      # 섹션 5 데이터 범위
        metrics_start=metrics_start,
        insights_start=insights_start,
    )
```

#### 신규 메서드

| 메서드 | 용도 |
|--------|------|
| `_delete_existing_charts()` | 대시보드 시트의 기존 차트 전체 삭제 |
| `_create_charts(...)` | 6개 차트 순차 생성 |
| `_build_column_chart_spec(...)` | 세로 막대 차트 스펙 생성 |
| `_build_line_chart_spec(...)` | 라인 차트 스펙 생성 |
| `_build_pie_chart_spec(...)` | 파이 차트 스펙 생성 |
| `_build_bar_chart_spec(...)` | 가로 막대 차트 스펙 생성 |

### 3.3 기존 코드 영향

- 기존 6개 섹션의 데이터 작성 로직: **변경 없음**
- 기존 포맷팅(색상, 숫자 포맷): **변경 없음**

## 4. 논의사항

### 4.1 차트 데이터 참조 방식

**옵션 A: 기존 테이블 데이터 직접 참조 (권장)**
- 섹션 5의 테이블 행을 차트의 sourceRange로 직접 참조
- 장점: 별도 데이터 영역 불필요, 데이터 중복 없음
- 단점: 데이터 행 위치가 동적이므로 차트 생성 시 정확한 행 인덱스 전달 필요

**옵션 B: 차트 전용 데이터 영역 생성**
- 차트용 데이터를 별도 영역(예: AA열 이후)에 복사
- 장점: 차트 데이터가 테이블 구조에 독립적
- 단점: 데이터 중복, 관리 복잡도 증가

### 4.2 섹션 3, 4의 데이터 참조 문제

섹션 3(투자 지표)과 섹션 4(매매 인사이트)는 **지표명-값** 2컬럼 구조로, 행 위치가 소섹션 구분자·제목 행에 의해 동적으로 변한다. 이 데이터를 차트로 참조하려면:

**옵션 A: 소섹션 행 위치를 반환값으로 추적**
- `_write_investment_metrics()`가 각 소섹션의 시작/끝 행을 반환
- 차트 생성 시 이 위치를 사용

**옵션 B: 차트용 데이터를 별도 행에 요약**
- 파이 차트, 바 차트에 필요한 데이터만 별도 영역에 작성
- 해당 영역을 차트가 참조

### 4.3 차트 개수 조절

6개 차트 모두를 한 번에 구현할 수도 있고, 우선순위에 따라 단계적으로 추가할 수도 있다.

**우선순위 제안:**
1. **P0 (필수)**: 차트 1 (월별 실현손익), 차트 2 (승률·수익률 추이) - 추세 파악에 가장 핵심
2. **P1 (권장)**: 차트 3 (계좌별 투자비중), 차트 5 (손익비·PF 추이)
3. **P2 (선택)**: 차트 4 (요일별 성과), 차트 6 (수익 Top 10)

### 4.4 차트 스타일 커스터마이징

Google Sheets API의 `BasicChartSpec`으로 제어 가능한 스타일:
- 차트 제목, 축 제목
- 범례 위치 (BOTTOM, RIGHT, NONE 등)
- 시리즈별 색상
- 축 포맷 (숫자, 백분율 등)

제어 **불가능**한 스타일:
- 조건부 막대 색상 (양수=파랑, 음수=빨강)은 BasicChart에서 직접 지원하지 않음
  - 대안: 양수/음수 시리즈를 분리하여 2개 시리즈로 표현

### 4.5 API 할당량 고려

- 차트 생성은 `batchUpdate` API를 사용하므로, 6개 차트를 한 번의 batchUpdate 요청에 묶어 호출하여 API 호출 횟수를 최소화한다
- 차트 삭제도 마찬가지로 배치 처리
